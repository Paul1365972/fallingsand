# Client

Bevy app, display/input/UI only — it never drives game logic. States: `MainMenu → InGame`, with `Connecting → Playing` and a connection screen that clears once the first full tick batch applies; mid-game loss reuses it as a reconnecting overlay.

- **Rendering**: one GPU texture per chunk, re-uploaded only for dirty sub-rects, one quad each. Material id + shade resolve to color via a palette texture in a fragment shader — no per-pixel CPU loop.
- **Layered lighting**: the world (chunks/players/particles, render layer `WORLD_LAYER`) renders to an offscreen HDR texture via a world camera; a composite camera draws the sky behind, then a fullscreen quad that samples that texture and applies night darkness + torch/emissive lights as an alpha-preserving multiply. Air stays transparent, so lights only touch solid world and never brighten the sky (no torch vignette on stars). Both cameras share follow+zoom; bloom/tonemapping live on the composite; UI targets it via `IsDefaultUiCamera`.
- **Camera**: Noita-like virtual resolution ~424×242 cells. `Ctrl`+scroll zooms in discrete geometric steps (√√2 apart) between 0.5× and 2.0×, always including 1.0× (plain scroll selects the hotbar slot). Interest budgets are sized for the default zoom, so zooming out never grows the server's obligations.
- **Debug overlay** (F3): Minecraft-style two columns — left is world/player context (coords, chunk/region, world clock, player readout, cursor/selected material), right is performance/system (fps + frame-time, sim time + tick health, chunk/region counts, bandwidth, entities/bodies/particles, memory). Fixed-width fields; fast-changing counters are averaged over a ~1s window (uploads/bandwidth as per-second rates) so they read steadily. Server-side stats are embedded-single-player only. F3+G toggles the boundary/delta-rect visualizer; F3+N switches game mode.
- **Sky**: `Calendar::celestial()` in core is the single deterministic source — a native-2D "great wheel" cosmology where the rendered plane *is* the physics: on-screen overlap equals eclipse math by construction (a 3D celestial sphere cannot be flattened without render/physics splits). An elliptical track (1 × 0.62 sky units, ×133 cells) turns behind the flat world; night is the world blocking the sun. The hub floats ±0.30 over the 60-day year (four 15-day seasons; year 0 = noon, spring equinox), giving day length (8.1–15.9 h) and noon height in one mechanism. The moon laps the sun every 6.43 d, breathes ±20% in size and speed (5.87 d — supermoons, syzygies shifted ±0.2 d), and drifts radially between rails (1 ± 0.48, 4.652 d) — the radial gap is the 2D analogue of inclination, setting each lap's miss distance. Solar eclipse = the moon shrouding the sun; occlusion is the exact covered-area fraction, so total/annular/partial/miss fall out of geometry. Lunar eclipse = the **Shade**, an invisible crimson-glass disc (radius 0.125) on a mirrored outer track (1.06), up only when the sun is down; the moon caught behind it turns blood-red. Its existence is implied, never drawn — inferred from the bites it takes. Epoch offsets keep age 0 unremarkable; golden-ratio phase steps keep timing irregular: one visible eclipse per ~7 days, windows ≈1–2 real minutes. `CelestialState` (positions, illumination, occlusion/shadow, star visibility, sidereal time, `light` budget) is f32 and **client-visual only**; server gameplay gates must use integer calendar math (`season()`, `day_of_year()`). Sun/moon/stars are pixel-art PNGs (`assets/sky/`) at one texel per world cell, fixed disc sizes; shaders add bloom, horizon redness, phase terminator, and eclipse blood tint (applied in day and night branches, graded by umbra depth, craters showing through). Stars fade on sun depression (−1° → −16°), brightest first, washed by moonlight, and reveal whenever `light` drops below ~0.3 (any eclipse darkness); the starfield drifts with sidereal time. Night darkness and horizon haze are driven by `light`.
- **Inventory UI**: `E` toggles a full-screen overlay (player slot grid + hotbar; side panel is the crafting list or the creative item palette). Drag & drop reflects the server-authoritative cursor with no prediction (left pick/place/swap, right half/one, shift-left quick-move, backdrop-click drops to world); tooltips on hover. The hotbar (slots 0..9) is always visible; digits/scroll select the slot (`Ctrl`+scroll zooms the camera), `[`/`]` size the dig-place brush. World input is suppressed while the overlay or chat is open. Dropped items render as bobbing, interpolated swatches on `WORLD_LAYER`. See [Inventory.md](Inventory.md).
- **Input**: `input.rs` owns every raw key/mouse read — one hardcoded keymap, one input context (chat > paused > inventory > gameplay) replacing scattered open-flag checks, per-frame sampling that latches held state, queues actions, and blocks pointer presses starting over UI, flushed as one `InputFrame` per fixed tick so edges survive fixed-step aliasing. Local hotkeys are `LocalAction` messages; gameplay systems read `InputHeld`.
- **Keys**: WASD/arrows + Space move/jump, S crouch, double-tap Space creative flight, LMB/RMB dig/place, digits/scroll hotbar, `[`/`]` brush, E inventory, Enter chat, Esc back, F2 screenshot, F11 fullscreen, F3 overlay (see above).
