# Client

Bevy app, display/input/UI only — it never drives game logic. States: `MainMenu → InGame`, with `Connecting → Playing`; the connection screen clears once the first full tick batch applies and doubles as the reconnect overlay.

- **Rendering**: one GPU texture per chunk, re-uploaded only for dirty sub-rects, one quad each. Material id + shade resolve to color via a palette texture in a fragment shader — no per-pixel CPU loop. Players have no sprite: their flesh cells arrive in chunk deltas and render as world cells (the `flesh` shade pattern is the character art); each player keeps an invisible anchor entity at the replicated cell pose, cell-snapped and uninterpolated, carrying the nametag and burning flames and serving as the camera target — the camera's exponential smoothing is what keeps whole-cell steps readable.
- **Layered lighting**: the world renders to an offscreen HDR texture (`WORLD_LAYER`); a composite camera draws the sky behind, then a fullscreen quad applies night darkness + emissive lights as an alpha-preserving multiply — air stays transparent, so lights never brighten the sky. Both cameras share follow+zoom; bloom/tonemapping live on the composite; UI targets it via `IsDefaultUiCamera`.
- **Camera**: Noita-like virtual resolution ~424×242 cells. `Ctrl`+scroll zooms in geometric steps (√√2 apart) between 0.5× and 2.0×, always including 1.0×. Interest budgets are sized for the default zoom, so zooming out never grows server obligations.
- **Debug overlay** (F3): Minecraft-style two columns — left world/player context, right performance/system. Fast-changing counters are averaged over ~1 s; server-side stats are embedded-single-player only. F3+G toggles the rect visualizer; F3+N switches game mode.
- **Sky**: `Calendar::celestial()` in core is the single deterministic source — a native-2D "great wheel" cosmology where the rendered plane *is* the physics: on-screen overlap equals eclipse math by construction. An elliptical sun track behind the flat world gives night (the world blocks the sun), day length (8.1–15.9 h), and noon height in one mechanism over the 60-day year (four seasons). The moon laps the sun, breathes in size and speed, and drifts radially between rails — the 2D analogue of inclination, setting each lap's miss distance. Solar eclipse = the moon shrouding the sun, occlusion as exact covered-area fraction (total/annular/partial fall out of geometry). Lunar eclipse = the **Shade**, an invisible crimson-glass disc on a mirrored outer track, up only when the sun is down; the moon caught behind it turns blood-red — implied, never drawn. Golden-ratio phase steps keep eclipses irregular (~1 visible per 7 days). `CelestialState` (positions, illumination, occlusion, star visibility, sidereal time, `light`) is f32 and **client-visual only** — server gameplay gates use integer calendar math (`season()`, `day_of_year()`). Sun/moon/stars are pixel-art PNGs (`assets/sky/`) at one texel per world cell; shaders add bloom, horizon redness, phase terminator, and eclipse blood tint. Stars fade on sun depression, wash under moonlight, and reveal when `light` < ~0.3; the starfield drifts with sidereal time. Explore with [skysim.html](skysim.html).
- **Inventory UI**: see [Inventory.md](Inventory.md) — `E` overlay, server-authoritative drag & drop, always-visible hotbar. World input is suppressed while the overlay or chat is open.
- **Input**: `input.rs` owns every raw key/mouse read — one hardcoded keymap, one input context (chat > paused > inventory > gameplay), per-frame sampling that latches held state, queues actions, and blocks pointer presses starting over UI, flushed as one `InputFrame` per fixed tick so edges survive fixed-step aliasing. Local hotkeys are `LocalAction` messages; gameplay systems read `InputHeld`.
- **Keys**: WASD/arrows + Space move/jump, S crouch, double-tap Space creative flight, LMB/RMB dig/place, digits/scroll hotbar, `[`/`]` brush, E inventory, Enter chat, Esc back, F2 screenshot, F11 fullscreen, F3 overlay.
